Why developers reach for Laradock

Any PHP Version

Run any version from 5.6 to 8.5. Set PHP_VERSION in .env, rebuild, and you're on it.

100+ Ready-made Services

Databases, caches, queues, search engines, and more, all pre-configured and waiting.

All-in-One Dev Shell

Run Artisan, Composer, Node, and any CLI inside workspace container, nothing on host.

Pick Your Database

MySQL, PostgreSQL, MariaDB, MongoDB, Redis, and many others, ready to switch on.

Framework-Agnostic

Works with Laravel, Symfony, WordPress, Magento, Drupal, or plain PHP.

Local AI, Built In

Run LLMs and vector search locally with Ollama, LiteLLM, pgvector, Qdrant, no cloud bills.

Toggle Services On Demand

Start only what a project needs with docker compose up, and stop them easily.

One Environment Everywhere

Identical setup on Linux, macOS, and Windows, so your team shares the same stack.

Configure From One File

Every service ships pre-configured; override any setting with 1 line in .env, always wins.

Web Server Ready

NGINX, Apache, and Caddy come pre-configured to serve your code out of the box.

One or Many Projects

Run a dedicated Laradock per project, or share a single setup across all of them.

Deploy to Production

Turn your dev stack into a hardened image with ./laradock ship, then deploy anywhere.
